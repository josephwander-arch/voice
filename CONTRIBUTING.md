# Contributing to voice

Thanks for your interest in contributing.

## Reporting bugs

Open an issue using the bug template. Include:
- Your OS and voice version
- What you tried
- What happened vs what you expected
- Logs from C:\CPC\logs\ (or your platform equivalent) if relevant

## Suggesting features

Open an issue using the feature request template. Explain the problem first,
then your suggested solution.

## Pull requests

Right now voice is mostly maintained by me with AI agents. PRs from the
community are welcome but please open an issue first to discuss before doing
significant work — I'd hate for you to spend time on something that doesn't
fit the roadmap.

Branch protection on `main` requires status checks (CodeQL) to pass before merge.

## Code of conduct

Be kind. Don't be a jerk. Disagreements are fine, personal attacks are not.
